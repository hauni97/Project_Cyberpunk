# Contributing to PROJECT CYBERPUNK

Thank you for your interest in contributing! This document provides guidelines for contributing to this project.

## Project Vision

PROJECT CYBERPUNK aims to create a discreet, voice-controlled penetration testing platform that combines modern AR technology with established security tools - all for **authorized security assessments only**.

## How to Contribute

### For New Contributors

1. **Read the Documentation**: Familiarize yourself with the [README](README.md) and documentation in `/docs`
2. **Check Issues**: Look for issues tagged with `good first issue` or `help wanted`
3. **Introduce Yourself**: Open a discussion in the "Introductions" section
4. **Start Small**: Begin with documentation improvements, bug reports, or small fixes

### Types of Contributions We Need

- **Documentation**: Improvements, translations, tutorials
- **Hardware**: Testing with different devices, compatibility reports
- **Code**: Implementation of features from the roadmap
- **Security**: OPSEC audits, vulnerability reports
- **Testing**: Field testing in controlled environments
- **Design**: UI/UX for HUD overlays, system sounds

## Contribution Process

### 1. Before You Start

- Check if someone else is already working on it (Issues/Discussions)
- For major changes, open an issue first to discuss your approach
- Make sure your contribution aligns with the project goals

### 2. Fork & Branch
```bash
# Fork the repository on GitHub (click "Fork" button)
# Clone your fork
git clone https://github.com/YOUR-USERNAME/project-cyberpunk.git
cd project-cyberpunk

# Create a feature branch
git checkout -b feature/your-feature-name
```

### 3. Make Your Changes

- Follow the existing code style
- Add comments for complex logic
- Update documentation if needed
- Test your changes thoroughly

### 4. Commit Guidelines

Use clear, descriptive commit messages:
```
Good:
"Add wake word detection using Porcupine"
"Fix Bluetooth latency issue in audio pipeline"
"Update hardware compatibility list"

Bad:
"Update"
"Fix stuff"
"asdf"
```

### 5. Submit a Pull Request

1. Push to your fork: `git push origin feature/your-feature-name`
2. Go to the original repository on GitHub
3. Click "Pull Request" â†’ "New Pull Request"
4. Select your branch
5. Fill out the PR template with:
   - What you changed
   - Why you changed it
   - How to test it

### 6. Code Review

- The maintainer will review your PR
- Be responsive to feedback
- Make requested changes
- Once approved, it will be merged

## What We Don't Accept

- Code that facilitates illegal activities
- Contributions without proper attribution
- Malicious code or backdoors
- Closed-source additions
- Features that compromise OPSEC
- Poor quality code without documentation

## Security Vulnerabilities

**DO NOT** open public issues for security vulnerabilities.

Instead, email: [Project.Cyberpunk@gmx.de] (or GitHub private vulnerability reporting)

Use GitHub's private security advisory feature
We will respond within 48 hours with next steps.
